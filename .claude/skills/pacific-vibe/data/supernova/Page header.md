# Page header

The header-page component is a typographic header component that defines the spacing and dimension relationship between a page title and subtitle. This is a common typographic lockup that’s used frequently in our funnels, flows, and full page takeovers.

## Accessibility

Elements should be labeled properly to ensure proper function of assistive technology such as screen readers, screen magnifiers, and speech recognition software, used by people with disabilities. Refer to the WCAG guidelines for detailed accessibility guidelines.

Focus order refers to the logical sequence in which interactive elements (such as buttons, links, and form inputs) receive focus when a user navigates through a page using the keyboard (typically the Tab key). For screen reader accessibility, the focus order is crucial because it dictates how users perceive and interact with the content.

## Usage

The Page Header is a prominent UI element at the top of a page or screen that introduces the content below it. It serves as a visual anchor and provides users with essential information about the page’s purpose. Page headers typically include a combination of the following elements:
Title: The primary identifier of the page's content or purpose.
Subtitle (optional): Additional context or information to clarify or support the title.
Page headers help establish hierarchy and set user expectations, ensuring clear navigation and understanding.
Considerations
When using the Page Header component, it’s important to take into consideration the following guidelines.
Common use cases
Page headers are most commonly used in our funnels and flows such as onboarding, applications, full-page loading, account creation, and editing/reviewing funnels.
Content limitations
Page headers should be clear, concise, and informative. The title should focus on the main action or purpose of the page, while the subtitle provides additional context or instructions.
Title best practices
Keep it short and scannable (~37 characters)
375+ viewport sizes: 3 lines max (2 preferred)
320-375 viewport sizes: 4 lines max (2-3 preferred)
Ideally, the title should be one sentence. If that’s the case, don’t use a period at the end (but do use a question mark if it’s a question). If the title is more than one sentence, use punctuation as needed for clarity.
Subtitle best practices
Use to clarify the page’s purpose or offer brief instructions.
Keep it informative but concise (~108 characters).
375+ viewport sizes
320-375 viewport sizes
