# Task list

A Task List component displays a list of tasks or items, often with checkboxes or status indicators, allowing users to track progress, mark tasks as complete, or take action on individual items. It's commonly used in product onboarding and account setup experiences.

## Accessibility

Elements should be labeled properly to ensure proper function of assistive technology such as screen readers, screen magnifiers, and speech recognition software, used by people with disabilities. Refer to the WCAG guidelines for detailed accessibility guidelines.

## Usage

Task lists help users track progress, complete actions, and stay oriented during multi-step workflows. Use them to clearly outline a set of tasks or steps that need user attention, especially when those tasks contribute to account setup, security, or feature unlocks.
We support two types of task lists:
List layout: a full-width, vertical list with more breathing room, ideal for onboarding flows or full-page experiences.
Card layout: a condensed version that appears within a card on the home screen or dashboard, optimized for quick glances and efficient use of space.

**When to use:** To guide users through onboarding or account setup
To highlight incomplete actions that unlock features or improve account performance
To track progress through a defined sequence of tasks (e.g., loan application, direct deposit setup)
To visually encourage engagement with optional but valuable tasks

**Don't use when:** When only one action is required (use a banner, modal, or button instead)
When the order of tasks doesn’t matter and isn’t part of a broader flow
When tracking is passive or informational (use a status pill or progress meter instead)

Choosing a layout
Use the list layout
In dedicated funnels or full-page experiences
When tasks are the primary focus of the screen
When there’s enough space to show detailed descriptions or supporting actions
For onboarding, account setup, or multi-step processes
Use the card layout
On overview or dashboard pages where space is limited
When tasks are secondary to the main content, but still need visibility
To summarize a user’s progress and highlight a few key tasks
When used alongside other cards or modules in a grid or stack

Common use cases
Account setup checklist (e.g., link external accounts, set up security)
Direct deposit setup steps
Credit card activation and benefit highlights
Tax preparation task list
Personalized action center on the dashboard (card layout)

**Do:**
- Use clear, action-oriented labels for each task.
- Show completion states (glyphs or status pills).
- Use the card layout only when space is limited or multiple modules are present.

**Don't:**
- Don’t overload the list with more than 5–7 tasks—group or stage if necessary.
- Don’t include status in the title or description.
- Don’t truncate copy or use more than one line for the task title. Copy should be short and direct.
