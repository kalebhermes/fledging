# Tabs

Tabs allows users to toggle between different views or subsets within an application or webpage. It highlights the active selection and is best suited for closely related, mutually exclusive options that share the same space.

Types of tabs

In pacific, tabs are categorized in 2 types -

Primary tabs: Used to navigate between the main content destinations within an application. They are positioned at the top of the content pane, directly below the top app bar, ensuring prominence and easy discoverability.

Secondary tabs: Secondary tabs are used to further organize and separate related content within a specific section of the application. They are placed within the content area, to provide additional navigation and reinforce the content hierarchy.

## Accessibility

Screen reader labeling for tabs should align with the visible label of the tab. Additional details can be included in the screen reader label when needed to provide clarity and reduce potential confusion. You should also include the tab's position in the set (e.g., "Option 1 of 5"), to help users understand the total number of tabs in the menu.

The focus initially appears on the first interactive element, signaling to the user that it is selected. From there, the user can continue navigating through the remaining interactive elements within the tab menu until all available items are navigated.

Keyboard:
- Tab: Move to the next actionable on the screen
- Space or Enter: Activate a tab

## Usage

**When to use:** Organize Related Information: Use tabs to categorize and display related groups of information, making it easy for users to switch between different sections.
Provide Quick Access: Tabs are ideal when users need to access multiple categories or views of content within the same page or context without losing their place.
Use for Multiple Views: Tabs are perfect for situations where you need to display different views of the same content or context.

**Don't use when:** Avoid for Sequential Steps: Do not use tabs to guide users through a sequence of actions. For multi-step processes, use pagination or a progress bar instead.
Overloading with Too Many Tabs: Avoid using more than six tabs. Too many tabs can overwhelm users and lead to a cluttered interface. Use a dropdown menu or accordion for more options.
Don’t Use for Unrelated Content: Tabs should not be used for displaying completely unrelated content. If the content does not logically belong together, it’s better to explore other navigation solutions like side menus.

**Do:**
- Keep tab labels short and concise, ideally one or two words, to ensure clarity and easy navigation.
- Use tabs to categorize related groups of information, allowing users to easily switch between different sections within the same context.
- Use a performance range to present filtering options in graphs, as it allows users to adjust the displayed data within a specific range without cluttering the interface.

**Don't:**
- Avoid using lengthy text or truncate tab labels, as this can clutter the interface and hinder user understanding
- Avoid using tabs to define steps in a process or workflow. Instead, use pagination for multi-step tasks to guide users through a linear flow.
- Use tabs for filtering options in graphs. Tabs are better suited for categorizing distinct sections of content, not for adjusting data ranges or continuous filtering.
